11 # @file BasisSettings.cmake
12 # @brief Default project-independent settings.
13 #
14 # This module defines global CMake constants and variables which are used
15 # by the BASIS CMake functions and macros. Hence, these values can be used
16 # to configure the behavior of these functions to some extent without the
17 # need to modify the functions themselves.
18 #
19 # @note As this file also sets the CMake policies to be used, it has to
20 # be included using the @c NO_POLICY_SCOPE in order for these policies
21 # to take effect also in the including file and its subdirectories.
22 #
23 # @attention Be careful when caching any of the variables. Usually, this
24 # file is included in the root CMake configuration file of the
25 # project which may also be a module of another project and hence
26 # may overwrite this project's settings.
27 #
28 # @attention Keep in mind that this file is included before any other
29 # BASIS module. Further, project-specific information such as
30 # the project name are not defined yet.
31 #
32 # @ingroup BasisSettings
33 ##############################################################################

371 ## @brief Whether BASIS shall use target UIDs.
372 #
373 # If this option is OFF, target UIDs are idential to the target names
374 # given as arguments to the "basis_add_*" functions.
375 #
376 # The target UIDs ensure that no name conflict between the targets
377 # of this project and those of an external library which are imported
378 # occurs. Another reason for using these target UIDs is to avoid
379 # target name conflicts between modules or subprojects which may
380 # be developed by different teams.
381 #
382 # The downside of using target UIDs is, however, a slower configuration
383 # of the build system because every target name must be mapped to its
384 # target UID and possibly vice versa. Moreover, the use of target UIDs
385 # is less intuitive for those new to BASIS but experienced with CMake.
386 basis_set_if_empty (BASIS_USE_TARGET_UIDS OFF)
387 
388 ## @brief Whether BASIS shall use fully-qualified target UIDs.
389 #
390 # If this option is OFF, the namespace of the top-level BASIS project is
391 # not prepended to the actual CMake build target names.
392 #
393 # For example, instead of the fully-qualified target UID
394 # "@PROJECT_NAME_L@.target", the CMake target name will simply
395 # be "target". Only when the target is referenced from another project,
396 # the fully-qualified target UID is usually required.
397 basis_set_if_empty (BASIS_USE_FULLY_QUALIFIED_UIDS OFF)

472 ## @brief Enable the installation of scripted modules in site specific default directories.
473 #
474 # If this option is @c ON, scripted modules such as Python and Perl modules, in particular,
475 # are installed in the default installation directories for site packages of the respective
476 # interpreter. This means that these modules may be installed outside the installation
477 # root directory as specified by the @c CMAKE_INSTALL_PREFIX. When this option is set to
478 # @c OFF, all modules are installed in subdirectories of the @c CMAKE_INSTALL_PREFIX.
479 # These directories may have to be added to the search path for modules manually as they
480 # might not be in the default search path of the respective interpreter.
481 #
482 # The installation directories for public modules which are intended for external use
483 # can further be set using the -D option of CMake or be modified by editing the respective
484 # advanced CMake cache variables named <tt>INSTALL_&lt;LANG%gt;_SITE_DIR</tt>.
485 #
486 # @note Even though it is more convenient for the user of a package to have the modules
487 # being installed in the default directories, this option is set to @c OFF by default.
488 # The reasons are that it is in first place expected that the installation will copy
489 # files only to directories within the @c CMAKE_INSTALL_PREFIX. Moreover, it is not
490 # guaranteed that the user has write permissions for the default site packages directories.
491 # Last but not least, when installing public modules located in the @c PROJECT_LIBRARY_DIR
492 # source directory, BASIS does not set a default module @c PREFIX which reduces the risk
493 # of overwriting existing modules of other packages. If the developer of a BASIS package
494 # was not thorough enough and did not follow the guidelines, setting this option to @c ON
495 # has the potential risk of overwriting other packages' modules. Therefore,
496 # modules are only installed in system default locations if explicitly requested.
497 option (BASIS_INSTALL_SITE_PACKAGES "Enable the installation of scripted modules in site specific default directories." OFF)
498 mark_as_advanced (BASIS_INSTALL_SITE_PACKAGES)
