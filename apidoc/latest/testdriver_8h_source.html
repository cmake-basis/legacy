1 /**
2  * @file testdriver.h
3  * @brief Default test driver include file.
4  *
5  * This file is specified as INCLUDE argument to the create_test_sourcelist()
6  * command of CMake which generates the code of the test driver. Such test
7  * driver is used, in particular, to run a test which generates an output image.
8  * The resulting image can then be compared by the test driver to one or more
9  * baseline images. Note the difference to plain non-image processing based
10  * unit tests. These shall make use of the unit testing frameworks included
11  * with BASIS instead (see test.h for a C++ unit testing framework).
12  *
13  * This file in particular declares the functions which are used to parse
14  * the command-line arguments of the test driver and those which are used by
15  * the code fragments defined in the files testdriver-before-test.inc and
16  * testdriver-after-test.inc.
17  *
18  * Currently available test driver implementations included by this file are:
19  * - testdriver.hxx
20  * - testdriver-itk.hxx

353  * @brief Generate list of names of baseline files from a given template filename.
354  *
355  * The list of baseline file names is generated from the template filename using
356  * the following algorithm:
357  * -# Strip the file name suffix.
358  * -# Append a suffix containing of a dot (.) and a digit, i.e., .x
359  * -# Append the original file name suffix.
360  * It the file exists, increment x and continue.
361  *
362  * Additionally, if a file @p filename_template exists, it is the first
363  * element in the resulting list.
364  *
