56 /**
57  * @brief Provides information about executable build targets.
58  *
59  * Within source code of a BASIS project, other SBIA executables are called
60  * only indirectly using the build target name which must be fixed and unique
61  * within the lab. The name of the output executable file of these targets may
62  * however vary and be finally set by BASIS, for example, depending on
63  * whether the project is build as part of a superproject or not. Therefore,
64  * each BASIS CMake function may adjust the output name in order to resolve
65  * name conflicts with other targets or SBIA executables.
66  *
67  * The idea is that a target name is supposed to be stable and known to the
68  * developer as soon as the target is added to a CMakeLists.txt file, while
69  * the name of the actual executable file is not known a priori as it is set
70  * by the BASIS CMake functions during the configure step. Thus, the developer
71  * should not rely on a particular name of the executable file. Instead, they
72  * can rely on the name of the corresponding build target which was chosen by
73  * themselves when adding the target to the build configuration.
74  *
75  * In order to get the actual file path of the built executable file, the
76  * function get_executable_path() is provided by the stdaux.h module.
77  * This function uses the static singleton instance of this class in order to
78  * map the given build target name to the name of the built and optionally
79  * installed executable. The code which initializes the required maps is
80  * generated automatically during the configuration of the build system.
81  *
82  * @sa exepath()
83  */

203 /**
204  * @brief Get absolute path of executable file.
205  *
206  * This function determines the absolute file path of an executable. If no
207  * arguments are given, the absolute path of this executable is returned.
208  * If the command names a known executable build target, the absolute path to
209  * the corresonding built (and installed) executable file is returned.
210  * Otherwise, the named command is searched in the system @c PATH and its
211  * absolute path returned if found. If the executable is not found, an
212  * empty string is returned.
213  *
214  * @todo This function currently makes use of the which command implemented
215  * in Python and called as subprocess in order to search a command
216  * in the system @c PATH. This which command is part of BASIS and
217  * can also be used on Windows. However, a native C++ implementation
218  * would be desireable.
219  *
220  * @param [in] name Name of command or @c NULL.
221  * @param [in] targets Structure providing information about executable targets.
222  *
223  * @returns Absolute path of executable or an empty string if not found.
224  * If @p name is @c NULL, the path of this executable is returned.
225  *
226  * @sa exename()
227  * @sa exedir()
228  */
